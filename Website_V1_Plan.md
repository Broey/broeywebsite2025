# Broey. Website V1 Plan

## Purpose

The Broey. website will be the central hub for Broey music, merch, visuals, links, contact opportunities, and future creative projects.

The goal of V1 is not to build a massive website. The goal is to create a polished, mobile-friendly, SEO-friendly home base that makes Broey feel official and gives visitors an easy path to listen, follow, shop, or contact.

## Core Goals

1. Make it easy for people to listen to Broey music.
2. Improve search visibility for Broey, broeybeats, releases, merch, and related content.
3. Provide one clean landing page for social links, streaming links, merch, and contact.
4. Create a branded experience that feels more intentional than Linktree or a generic artist template.
5. Build a foundation that can expand later into releases, press, plugins/tools, sample packs, and fan/community features.

## Mobile Experience Requirements

Mobile is a priority. Most visitors will likely come from social links, streaming apps, or mobile search.

### Mobile Rules

* Site must load quickly.
* Main CTA must be visible near the top.
* No important action should depend on hover.
* Buttons should be thumb-friendly.
* Body copy should be short.
* Text must have strong contrast.
* Navigation should be simple.
* Streaming links should be easy to tap.
* Merch cards should be clean and readable.
* Forms should be easy to complete on a phone.

### Mobile Header

Recommended structure:

```text
Broey.        Menu
```

### Mobile CTA Priority

## Desktop Experience Requirements

Desktop should feel more immersive and exploratory.

Desktop can include:

* Subtle hover states
* Release card motion
* Ambient visual effects
* Signal/waveform details
* Larger hero visuals
* Interactive archive filtering
* Floating or sticky elements, if tasteful

Desktop should still remain clear and fast.

## Interaction Guidelines

The site should be fun to explore, but interaction should never block usability.

### Good Interactions

* Release cards lift slightly on hover.
* Cover art has subtle tilt or glow.
* Buttons have clear hover/tap states.
* Active links glow or shift color.
* Section transitions fade in smoothly.
* Visualizer/waveform dividers animate subtly.
* Merch cards reveal a quick CTA.
* Music archive can filter by release type.

### Avoid

* Auto-playing music
* Heavy glitch effects
* Slow page transitions
* Hidden navigation
* Tiny tap targets
* Animations that delay content
* Hover-only streaming links on mobile

## Recommended Build Direction

### Preferred Technical Direction

Use a modern, SEO-friendly frontend with a lightweight commerce integration.

Recommended options:

1. Next.js + Tailwind + Vercel + Shopify
2. Astro + Tailwind + Netlify/Vercel + Shopify
3. Framer/Webflow + Shopify link/embed for a faster no-code build

### Best Long-Term Option

```text
Next.js + Tailwind + Vercel + Shopify
```

Why:

* Strong SEO
* Fast performance
* Easy deployment
* Flexible design
* Scales well
* Can add release pages, blog, tools, and plugins later

### Fastest Practical Option

```text
Framer or Webflow + Shopify
```

Why:

* Faster to design visually
* Less coding overhead
* Good for launching quickly
* Can still look professional
